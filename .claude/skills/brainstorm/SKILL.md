# /brainstorm — Идея → Дизайн → Выполнение

Один скилл для полного цикла: обсуждение → дизайн → план → автономное выполнение.

## Философия

- **Одна точка согласования** — после показа дизайна+плана. Дальше автономно
- **Вопросы только ДО плана** — не спрашивать разрешение на каждый шаг
- **Skills-first** — НИКОГДА не писать свои скрипты если есть скилл

## Маршрутизация (автоматическая)

ИИ сам определяет режим по задаче:

| Признаки | Режим | Что делать |
|----------|-------|-----------|
| 1-2 объекта, очевидная реализация | **express** | Кратко озвучить план → выполнить скиллами |
| 3-5 задач, нужен дизайн | **standard** | Полный brainstorm → план → выполнить |
| 6+ задач, архитектурное | **full** | Brainstorm → plan → субагенты с ревью |

## Процесс

### Фаза 1 — Понимание (диалог с пользователем)

Загрузи контекст:
```
rlm_route_context(query="project architecture current work")
```

Для БСП-связанных задач:
```
ssl_search(query="<ключевые слова>")
```

Задавай уточняющие вопросы **по одному** через AskUserQuestion:
- Предпочитай варианты ответа (не открытые вопросы)
- Фокус: зачем, кто использует, ограничения
- 1С-специфичные: конфигурация или CFE? Какие объекты? Нужна ли БСП?
- **Остановись когда задача ясна** — не спрашивай ради спрашивания

### Фаза 2 — Дизайн + план (ЕДИНСТВЕННАЯ точка согласования)

Покажи пользователю **одним сообщением**:

```markdown
## Дизайн: <название>

**Подход:** <1-3 предложения>

**Объекты:** <что создаём/меняем>

**Задачи:**
1. <задача> — скилл: /meta-compile
2. <задача> — скилл: /form-compile
3. <задача> — скилл: /subsystem-edit
...

**Риски:** <если есть>

Выполняю?
```

Масштабируй под сложность:
- **express**: 3-5 строк, без секции рисков
- **standard**: полный формат выше
- **full**: + альтернативные подходы с trade-offs, + секция «Отклонённые варианты»

**Жди ответа.** Пользователь скажет «ок» / скорректирует / откажет.

### Фаза 3 — Автономное выполнение

После «ок» — **выполняй молча, не спрашивая разрешений.**

#### Express/Standard режим: выполняй сам

Для каждой задачи из плана:
1. Вызови соответствующий скилл (Skill tool)
2. Если задача — BSL код, проверь через `1c-syntax-checker`
3. Коммить атомарно (одна задача = один коммит)

#### Full режим: субагенты

Для каждой задачи:
1. Dispatch implementer (Task tool) — дай полный текст задачи + список скиллов
2. Dispatch reviewer — проверь результат
3. При ❌ от reviewer — исправь и продолжай (не спрашивай пользователя)
4. При блокере (невозможно продолжить) — тогда спроси

Промпты субагентов: `.claude/skills/subagent-dev/implementer-prompt.md`, `.claude/skills/subagent-dev/reviewer-prompt.md`

### Фаза 4 — Отчёт

Один финальный отчёт:

```
Готово: <что сделано>
Файлы: <список изменённых>
Коммиты: <N коммитов>
Замечания: <если были проблемы в ревью>
```

## Сохранение артефактов

Для **standard** и **full** режимов сохрани:
- `openspec/changes/<feature>/design.md` — дизайн
- `openspec/changes/<feature>/tasks.md` — план с отметками [x]

Для **express** — не создавай файлы, не нужны.

## ЗАПРЕТЫ

- ❌ Спрашивать «можно выполнять задачу N?» после одобрения плана
- ❌ Спрашивать «эта часть выглядит правильно?» посекционно
- ❌ Писать PowerShell/BAT скрипты если есть скилл
- ❌ Писать XML руками если есть /meta-compile, /form-compile и т.д.
- ❌ Генерировать BSL из памяти без проверки через 1c-help и 1c-syntax-checker
