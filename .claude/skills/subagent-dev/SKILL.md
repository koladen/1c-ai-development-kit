# /subagent-dev — Автономное субагентное выполнение

Выполняет tasks.md через субагентов. Не спрашивает разрешений между задачами — работает до конца, отчитывается финально.

## Вход

`$ARGUMENTS` — путь к tasks.md или feature-name.

## Процесс

### 1. Подготовка

```bash
git rev-parse HEAD  # BASE_SHA
```

Прочитай tasks.md и design.md. Сообщи:
```
Выполняю: <feature>, задач: N
```

### 2. Цикл по задачам

Для каждой задачи:

#### Implementer
```
Task(subagent_type="general-purpose", prompt=<implementer-prompt.md>)
```
Передай: текст задачи (НЕ путь), контекст проекта, зависимости, список скиллов.

#### Reviewer (один проход)
```
Task(subagent_type="general-purpose", prompt=<reviewer-prompt.md>)
```
Проверяет И соответствие спецификации, И качество кода в одном проходе.

#### При ❌ от reviewer
Отправь замечания обратно implementer-у. **Не спрашивай пользователя** — исправляй и продолжай. Максимум 2 итерации на задачу. Если не решается за 2 — запиши проблему и двигайся дальше.

#### Пропуск ревью
Для задач без BSL кода (чистые метаданные через /meta-compile, /subsystem-edit и т.д.) — reviewer не нужен. Валидация через соответствующий `-validate` скилл достаточна.

#### Отметка
В tasks.md: `[ ]` → `[x]`

### 3. Финальный отчёт

```
Готово: N/N задач
Коммиты: <список>
Проблемы в ревью: <если были>
Нерешённые замечания: <если есть>
```

## Когда спрашивать пользователя

Только при блокере: зависимость отсутствует, задача противоречива, невозможно продолжить. Всё остальное — решай сам.

## Когда НЕ использовать

- 1-3 задачи → выполняй напрямую скиллами (без субагентов)
- Нет tasks.md → сначала `/write-plan` или `/brainstorm`
