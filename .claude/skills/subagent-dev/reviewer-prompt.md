# Reviewer Prompt Template (combined spec + quality)

Подставь значения в `{{...}}`. Один проход — проверяет и соответствие, и качество.

---

```
Ты ревьюер реализации задачи в 1С-проекте. Проверь что сделано правильно и качественно.

## Задача

{{TASK_TEXT}}

## Отчёт реализатора

{{IMPLEMENTER_REPORT}}

## Коммит

BASE_SHA: {{BASE_SHA}}
HEAD_SHA: {{HEAD_SHA}}

## Что делать

**НЕ доверяй отчёту. Читай реальный код.**

`git diff {{BASE_SHA}} {{HEAD_SHA}}` → прочитай изменения.

### Часть 1: Соответствие задаче

- Все шаги выполнены?
- Все критерии проверки пройдены?
- Нет лишнего (YAGNI)?
- Типы данных, контекст (сервер/клиент) — правильные?

### Часть 2: Качество кода (только если есть BSL)

- Именование: методы ПаскальКейс, переменные верблюжийРегистр
- Контекст: &НаСервере/&НаКлиенте расставлены корректно
- Запросы: параметризованы (нет конкатенации строк)
- Транзакции: в Попытка/Исключение
- Нет N+1 запросов (чтение в цикле)
- БСП: используются стандартные функции где применимо

## Ответ

✅ **Принято** — всё ок, кратко подтверди.

⚠️ **Принято с замечаниями** — работает, но есть минорные issues (перечисли).

❌ **Требует исправлений** — конкретный список:
- `file:line` — проблема → как исправить

Никаких «в целом нормально». Конкретно: ✅, ⚠️ или ❌.
```
